三、面向对象的三大特征
       封装、继承、多态
       多态：
       让有继承关系的类，通过对相同的成员方法的调用，产生不同的结果。多态也成为：同名异式
       1、语法
             - 抽象方法和抽象类
                a、抽象方法的定义，在定义抽象方法时必须加上关键词abstract.抽象方法没有方法体。
                b、抽象类定义时要求类的前面加上 abstract 来声明。
                c、一个类中如果含有抽象成员方法。则这个类必须被声明为抽象类。反之则不然。
                d、抽象类可以含有普通的成员方法和成员属性。
                e、抽象类不能够被实例化，抽象类是用来定义规范的。

             - 接口
                a、接口的定义 
                   interface  接口的名称{
                   }
                b、接口中不能有普通的成员方法和成员属性。只能有常量和抽象方法。
                c、接口中的抽象方法必须不能出现关键词 abstract。
                d、接口不能够被实例化，接口是用来定义规范的。

              - 抽象类和接口的继承
                 a、抽象类的继承
                       class 子类 extends 抽象类名称{
                       }  
                b、若子类继承了抽象类，子类必须重xie抽象类中所有抽象方法。
                c、类对于接口的继承。类对于接口的继承通过关键词
 implements。一个子类可以继承多个接口，多个接口之间以","进行间隔。
                 d、子类中继承接口，必须重xie接口中的所有抽象方法。 
                 e、类同时继承接口和抽象类（父类）  
                        class 类名称 extends 抽象类（父类） implements 
接口[,接口...]
                 f、接口对于接口的继承。extends
